# 파이썬 고급 스크랩핑

## HTTP 통신 기초 - cookie, session, request, response

* http 통신

  * Clinent -> Server 요청 
  * Server  -> Clinent 응답

  * 비연결 지향

    : 한 번 요청 보내고 응답하면 연결 끊김. 연결 비용 줄이기 위해.

  * 상태정보유지안함

    : 한 번 연결 끊기면 클라이언트 상태를 서버가 모름. 비용 줄이기 위해.

    

    -> 단점: 세션 정보 없으면 페이지 페이지 옮길 때마다 로그인해야하는 비효율적 상황

    -> 그래서 있는 것이 쿠키와 세션.

  * cookie, session 개념

    클라이언트에서 서버로 요청 보냄 -> 서버가 쿠키에 정보 실어서 클라이언트로 응답.

    ex) 자동로그인, 쇼핑몰 장바구니

    세션: 서버에서 저장. 클라이언트가 요청하면 세션 id 만료 여부 판단해서 응답.
